Foram realizados ajustes no primeiro microsserviço, mais especificamente na funcionalidade de 'DELETE', 'PUT (UPDATE). 
Em ambos, foi implementado um try/catch, para que quando o usuário tentar atualizar ou deletar um ID que foi deletado, uma mensagem amigável seja retornada, ao invés de lançar uma exception e crashar a execução.

Foi realizado um ajuste na funcionalidade 'GET (Consulta/Read)' para que, quando a lista de medicamentos estiver sendo percorrida, ao esbarrar num ID deletado, não lance uma exception e continue percorrendo a lista. Para isso foi utilizado o try/catch.

Foi implementado o segundo microsserviço, que permite a reserva/atualização de medicamentos de acordo com o ID. Foi utilizado a funcionalidade de UPDATE do primeiro microsserviço implementado (CRUD). Passamos no request o ID do medicamento e a quantidade a ser subtraída. O sistema realiza a subtração e retorna mensagens de acordo com a quantidade disponível do medicamento.
	Quando a quantidade disponível é igual a 0, emite um alerta de que o estoque do medicamento acabou.
	Quando a quantidade disponível está entre 1 e 3, emite um alerta de que o estoque do medicamente está prestes a acabar e atualiza a quantidade.
	Quando a quantidade disponível é maior que 3, emite um alerta de que a quantidade do medicamento foi atualizada com sucesso.
	
Foi implementado o esqueleto das telas que farão integração com o backend.
Foram feitas cinco telas: 
	home > A partir dela o usuário terá a oportunidade de acessar todas as outras funcionalidades (Inserir estoque, Consultar Estoque, Atualizar Estoque, Deletar Estoque)
	medicamento-inserir > Tela onde o usuário realizará os inputs para uso da funcionalidade de CREATE (Criar um novo medicamento)
	medicamento-consulta > Tela onde o usuário realizará os inputs para uso da funcionalidade de READ (Consultar estoque do medicamento)
	medicamento-atualizar > Tela onde o usuário realizará os inputs para uso da funcionalidade de UPDATE (Atualizar estoque do medicamento)
	medicamento-delete > Tela onde o usuário realizará os inputs para uso da funcionalidade de DELETE (Deletar estoque do medicamento)
